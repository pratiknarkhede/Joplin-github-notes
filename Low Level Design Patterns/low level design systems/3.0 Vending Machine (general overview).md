&nbsp;

**What does it DO?**

- - Sells items (snacks, drinks).
    - Accepts money (coins, maybe notes/cards later).
    - Lets users choose an item.
    - Gives the item if paid correctly.
    - Gives back change if needed.
    - Tells the user what's happening (display messages).
    - Needs to know when it's out of stock.
- **What physical/logical PARTS does it need?**
    
    - **The Machine itself:** The main controller.
    - **Items:** Things to sell (Cola, Chips, Candy). Each has a *name*, *price*, and maybe a *code* (like "A1", "B2").
    - **Storage/Inventory:** Shelves or slots holding the items. Needs to track *how many* of each item are left.
    - **Money Box:** Place to accept and store money. Needs to track the *current amount* inserted for a transaction. Maybe also total money collected.
    - **Selection Panel:** Buttons or keypad for the user to enter the item code.
    - **Dispenser:** The slot where the item drops out.
    - **Change Return:** The slot where extra money comes back.
    - **Display Screen:** To show price, messages ("Insert Coin", "Out of Stock").
    - **Internal State:** What is the machine *currently doing*? Is it waiting for someone? Waiting for money? Dispensing?
- **What ACTIONS can happen?**
    
    - User *selects* an item.
    - User *inserts* money.
    - Machine *dispenses* an item.
    - Machine *returns* change.
    - User *cancels* the transaction.
    - Admin *restocks* items.
    - Admin *collects* money.
- **How do these connect?**
    
    - The **Machine** *contains* the **Inventory**.
    - The **Machine** *uses* the **Money Box** (or logic for handling money).
    - The **Machine** reacts to **Selection Panel** input.
    - The **Machine**'s behaviour depends heavily on its **Internal State**.
    - Selecting an item (if available) changes the **State** to "waiting for money".
    - Inserting enough money changes the **State** to "ready to dispense".
    - Dispensing changes the **Inventory** count and changes the **State** back to "idle" (after giving change).

Let's refine the parts into potential classes and their responsibilities.

1.  **`VendingMachine` (The Main Class):**
    
    - *Responsibility:* Orchestrates the whole process. Knows the current state, manages transitions between states. Holds references to other major components.
    - *Potential Fields:* `currentState` (an object representing the state), `inventory` (an object managing items), `currentBalance` (money inserted for this transaction), `selectedItem` (the item the user picked).
    - *Potential Methods:* `selectItem(code)`, `insertMoney(amount)`, `dispenseItem()`, `cancelTransaction()`, `getChange()`.
2.  **`Item`:**
    
    - *Responsibility:* Represents a product being sold.
    - *Potential Fields:* `code` (String, e.g., "A1"), `name` (String, e.g., "Cola"), `price` (double or int).
3.  **`Inventory`:**
    
    - *Responsibility:* Manages the stock of items.
    - *Potential Fields:* A way to store items and their counts (e.g., a `Map<Item, Integer>` or `Map<String, Pair<Item, Integer>>` where the String is the item code).
    - *Potential Methods:* `getItem(code)`, `getQuantity(code)`, `isAvailable(code)`, `deductItem(code)`, `addItem(item, quantity)`.
4.  **`State` (The State Pattern):** This is a crucial design pattern for vending machines because their behaviour changes drastically based on what just happened.
    
    - *Responsibility:* Define an interface for actions. Each specific state (Idle, Accepting Money, Dispensing) will implement this interface differently.
    - *Potential Methods (in the interface):* `selectItem(vendingMachine, code)`, `insertMoney(vendingMachine, amount)`, `dispenseItem(vendingMachine)`, `cancelTransaction(vendingMachine)`.
    - *Concrete State Classes:*
        - `IdleState`: Allows item selection. Inserting money might be ignored or initiate selection.
        - `HasSelectionState` (or `AcceptingMoneyState`): An item has been selected. Allows money insertion. Allows cancellation. Might allow changing selection.
        - `DispensingState`: Item is being dispensed. No other actions allowed until done. Calculates change.
        - `SoldOutState`: The selected item is out of stock. Maybe allows selecting another item or cancelling.
        - `NoStockState`: The whole machine is empty (less common for LLD focus, but possible).
