The Observer Design Pattern is a behavioral design pattern that ==allows objects to notify other objects about changes to their state.==

It defines a one-to-many dependency between objects so that when one object changes its state, all its dependents are notified and updated automatically.

## Observer Design Pattern: The Core Idea

Think of a magazine subscription. You (the observer) subscribe to a magazine (the subject). Whenever the magazine releases a new issue,it notifies all its subscribers. The Observer Pattern works on a similar principle in software.

It establishes a one-to-many dependency between objects. A subject maintains a list of its dependents, called observers, and notifies them automatically of any state changes.

## Key Characteristics of the Observer Design Pattern

- **One-to-many dependency**: The pattern establishes a one-to-many dependency between objects, where one object (the subject) maintains a list of its dependents (observers).
- **Notification**: When the subject changes its state, it notifies all its observers, allowing them to react accordingly.
- **Decoupling**: The pattern decouples the subject from its observers, allowing them to operate independently.

## ==Components of the Observer Design Pattern==

1.  ==**Subject**: The subject is the object that maintains a list of observers and notifies them when its state changes.==
2.  ==**Observer**: The observer is the object that registers itself with the subject to receive notifications when the subject's state changes.==

&nbsp;

- **Interfaces:** `Product` defines methods for managing observers and notifying them. `CustomerObserver` defines the `update` method that observers must implement.
- **Concrete Subject:** `AmazonProduct` keeps track of its stock status and a list of observers. When its `setInStock` method is called with `true`, it notifies all observers.
- **Concrete Observer:** `Customer` stores an email address and implements the `update` method to send a notification.

```java
// Subject Interface
interface Product {
    void registerObserver(CustomerObserver observer);
    void removeObserver(CustomerObserver observer);
    void notifyObservers();
}





// Observer Interface
interface CustomerObserver {
    void update(Product product);
}





// Concrete Subject (Product)
class AmazonProduct implements Product {
    private String name;
    private boolean inStock = false;
    private List<CustomerObserver> observers = new ArrayList<>();

    // ... (Constructor, getters/setters)

    public void setInStock(boolean inStock) {
        this.inStock = inStock;
        if (inStock) {
            notifyObservers();
        }
    }

    @Override
    public void registerObserver(CustomerObserver observer) {
        observers.add(observer);
    }

    @Override
    public void removeObserver(CustomerObserver observer) {
        observers.remove(observer);
    }

    @Override
    public void notifyObservers() {
        for (CustomerObserver observer : observers) {
            observer.update(this);
        }
    }
}







// Concrete Observer (Customer)
class Customer implements CustomerObserver {
    private String email;

    // ... (Constructor, getters/setters)

    @Override
    public void update(Product product) {
        System.out.println("Email to " + email + ": Product " + product.getName() + " is back in stock!");
        // ... (Logic to send an actual email)
    }
}
```
