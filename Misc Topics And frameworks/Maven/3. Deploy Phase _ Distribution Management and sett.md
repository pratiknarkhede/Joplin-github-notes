**Distribution Management** is used to define how and where the project artifacts are distributed (deployed). It specifies the repositories for both releases and snapshots, allowing Maven to handle the deployment automatically during the `deploy` phase.  
<br/><br/>

```xml
<distributionManagement>
    <repository>
        <id>internal-repo</id>
        <url>http://repo.example.com/releases</url>
        <name>Internal Release Repository</name>
    </repository>
    <snapshotRepository>
        <id>internal-snapshots</id>
        <url>http://repo.example.com/snapshots</url>
        <name>Internal Snapshot Repository</name>
    </snapshotRepository>
</distributionManagement>

```

&nbsp;

- **`<repository>`**: Defines where to deploy release versions of the project.
- **`<snapshotRepository>`**: Defines where to deploy snapshot versions of the project (versions that are still under development).

**settings.xml Configuration**: The `settings.xml` file is crucial when the deployment requires credentials or specific configurations not stored in the POM (for security or flexibility reasons).

&nbsp;

```xml
<settings>
    <servers>
        <server>
            <id>internal-repo</id>
            <username>deployer</username>
            <password>{encrypted-password}</password>
        </server>
        <server>
            <id>internal-snapshots</id>
            <username>deployer</username>
            <password>{encrypted-password}</password>
        </server>
    </servers>
    <mirrors>
        <!-- Example: Configuring a mirror for all central repositories -->
        <mirror>
            <id>internal-mirror</id>
            <mirrorOf>central</mirrorOf>
            <url>http://repo.example.com/maven-central</url>
        </mirror>
    </mirrors>
    <profiles>
        <profile>
            <id>internal-repositories</id>
            <repositories>
                <repository>
                    <id>internal-repo</id>
                    <url>http://repo.example.com/releases</url>
                </repository>
            </repositories>
            <pluginRepositories>
                <pluginRepository>
                    <id>internal-plugins</id>
                    <url>http://repo.example.com/plugins</url>
                </pluginRepository>
            </pluginRepositories>
        </profile>
    </profiles>
    <activeProfiles>
        <activeProfile>internal-repositories</activeProfile>
    </activeProfiles>
</settings>

```

&nbsp;

&nbsp;

- **`<servers>`**: Defines credentials for accessing the repositories. Use `<id>` matching the repository ID in the POM.
- **`<mirrors>`**: Configures mirror repositories, redirecting all repository accesses to the specified URLs.
- **`<profiles>`**: Profiles for managing different configurations, repositories, or plugins based on environment or conditions.
- **`<activeProfiles>`**: Specifies which profiles are active by default.
