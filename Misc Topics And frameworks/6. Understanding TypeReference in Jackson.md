When deserializing JSON into Java objects using Jackson, you often encounter a common challenge related to Java's type erasure. Java's type system doesn't retain information about generic types at runtime. This means that while you can have a class like `List<Customer>`, at runtime, the JVM only sees a `List`, with no information about the `Customer` part. This is known as type erasure.

#### The Problem:

- When you try to deserialize a JSON array into a specific Java collection type (like a `List<Customer>`), Jackson needs to know the exact generic type to correctly instantiate and populate the collection.
- Due to type erasure, `List<Customer>.class` isn't a valid expression, and simply using `List.class` doesn't provide Jackson with enough information about the `Customer` type.

#### The Solution: `TypeReference`

- Jackson provides `TypeReference` as a way to work around the limitations of type erasure.
- `TypeReference` is an abstract class that you extend with an empty anonymous class, providing Jackson with the necessary generic type information.

#### How `TypeReference` Works:

- When you write `new TypeReference<List<Customer>>(){}`, you're creating an anonymous subclass of `TypeReference`. This subclass captures and retains the generic type information (`List<Customer>`) at runtime.
- Jackson uses this type information to correctly deserialize the JSON into a list of `Customer` objects, fully aware of the `Customer` type contained within the list.

#### Usage Example:

java

`ObjectMapper objectMapper = new ObjectMapper(); List<Customer> customers = objectMapper.readValue(jsonInput, new TypeReference<List<Customer>>(){});`

- In this example, `jsonInput` could be a `String`, `File`, `InputStream`, etc., containing the JSON data.
- The `objectMapper.readValue` method uses the `TypeReference` to understand that it should deserialize the JSON into a `List` of `Customer` objects.

### Summary

- `TypeReference` is a powerful tool provided by Jackson to preserve generic type information that would otherwise be lost due to type erasure.
- It enables precise deserialization of generic types, such as collections of specific object types, by providing Jackson with the necessary type information at runtime.
- This approach is essential when working with generic collections in Java and ensures that deserialization processes are type-safe and accurate.
