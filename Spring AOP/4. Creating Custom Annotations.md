Custom annotations allow you to mark methods or classes for specific AOP behavior without changing their code. Here's how to create and use a custom annotation:

&nbsp;

We define a custom annotation `@LogExecutionTime`.

```java
// Custom annotation
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface LogExecutionTime {
}
```

&nbsp;

We create an aspect that uses this annotation as a pointcut.

```java
// Aspect to use the custom annotation
@Aspect
@Component
public class LoggingAspect {

    @Around("@annotation(LogExecutionTime)")
    public Object logExecutionTime(ProceedingJoinPoint joinPoint) throws Throwable {
        long start = System.currentTimeMillis();

        Object result = joinPoint.proceed();

        long executionTime = System.currentTimeMillis() - start;

        System.out.println(joinPoint.getSignature() + " executed in " + executionTime + "ms");
        return result;
    }
}
```

&nbsp;

We can now use `@LogExecutionTime` on any method we want to log execution time for.

```java
@Service
public class ExampleService {

    @LogExecutionTime
    public void slowMethod() throws InterruptedException {
        Thread.sleep(1000);
    }
}
```

**Creating Custom Pointcuts**

Custom pointcuts allow you to define reusable pointcut expressions. Here's an example:

In below example:

1.  We define three pointcuts: `inServiceLayer()`, `anyPublicMethod()`, and `publicServiceMethod()`.
2.  `publicServiceMethod()` combines the other two pointcuts.
3.  We use `publicServiceMethod()` in our `@Before` advice.

```java
import org.aspectj.lang.annotation.Aspect;
import org.aspectj.lang.annotation.Before;
import org.aspectj.lang.annotation.Pointcut;
import org.springframework.stereotype.Component;

@Aspect
@Component
public class AuditingAspect {

    @Pointcut("within(com.example.service.*)")
    public void inServiceLayer() {}

    @Pointcut("execution(public * *(..))")
    public void anyPublicMethod() {}

    @Pointcut("inServiceLayer() && anyPublicMethod()")
    public void publicServiceMethod() {}

    @Before("publicServiceMethod()")
    public void auditMethodCall() {
        System.out.println("Public service method called");
    }
}
```

&nbsp;

**We can combine custom annotations and pointcuts for even more flexibility:**

In this example:

We define a custom `@Auditable` annotation that can take a custom message.

```java
@Target({ElementType.METHOD, ElementType.TYPE})
@Retention(RetentionPolicy.RUNTIME)
public @interface Auditable {
    String value() default "";
}

```

&nbsp;We create a pointcut that matches methods with this annotation.

We combine this with our `inServiceLayer()` pointcut.

Our advice uses the custom message if provided, or a default message if not.

```java
@Aspect
@Component
public class AuditingAspect {

    @Pointcut("within(com.example.service.*)")
    public void inServiceLayer() {}

    @Pointcut("@annotation(auditable)")
    public void auditableMethod(Auditable auditable) {}

    @Around("inServiceLayer() && auditableMethod(auditable)")
    public Object auditMethod(ProceedingJoinPoint joinPoint, Auditable auditable) throws Throwable {
        String auditMessage = auditable.value().isEmpty() 
            ? "Method audited: " + joinPoint.getSignature().toShortString()
            : auditable.value();
        
        System.out.println("Before: " + auditMessage);
        Object result = joinPoint.proceed();
        System.out.println("After: " + auditMessage);
        
        return result;
    }
}
```

&nbsp;We can now use `@Auditable` on any method in our service layer, optionally providing a custom audit message.

```java
@Service
public class UserService {
    @Auditable("User created")
    public void createUser(String username) {
        // User creation logic
    }

    @Auditable
    public void updateUser(String username) {
        // User update logic
    }
}
```

These techniques allow you to create more modular and reusable aspects, making your AOP code cleaner and more maintainable.
