&nbsp;

<span style="color: #d4d4d4;">A content delivery network (CDN) is a geographically distributed group of servers that work together to provide fast delivery of internet content. Generally, static files such as HTML/CSS/JS, photos, and videos are served from CDN.</span>

&nbsp;

## Why use a CDN?

Content Delivery Network (CDN) increases content availability and redundancy while reducing bandwidth costs and improving security.

<span style="color: #d4d4d4;">Serving content from CDNs can significantly improve performance as users receive content from data centers close to them and our servers do not have to serve requests that the CDN fulfills.</span>

## How does a CDN work?

<img src="../../../_resources/7168c3c09831fca3d0689054631e43fc.png" alt="7168c3c09831fca3d0689054631e43fc.png" width="730" height="405">

<span style="color: #d4d4d4;">In a CDN, the origin server contains the original versions of the content while the edge servers are numerous and distributed across various locations around the world.</span>

<span style="color: #d4d4d4;"><span style="color: #d4d4d4;">a CDN stores a cached version of its content in multiple geographical locations known as edge locations. Each edge location contains several caching servers responsible for content delivery to visitors within its proximity.</span></span>

<span style="color: #d4d4d4;"><span style="color: #d4d4d4;"><span style="color: #d4d4d4;">Once the static assets are cached on all the CDN servers for a particular location, all subsequent website visitor requests for static assets will be delivered from these edge servers instead of the origin, thus reducing the origin load and improving scalability.</span></span></span>

<span style="color: #d4d4d4;"><span style="color: #d4d4d4;"><span style="color: #d4d4d4;"><span style="color: #d4d4d4;">For example, when someone in the UK requests our website which might be hosted in the USA, they will be served from the closest edge location such as the London edge location. This is much quicker than having the visitor make a complete request to the origin server which will increase the latency.</span></span></span></span>

## Types

CDNs are generally divided into two types:

&nbsp;

### Pull CDNs

In a Pull CDN situation, the cache is updated based on request. When the client sends a request that requires static assets to be fetched from the CDN if the CDN doesn't have it, then it will fetch the newly updated assets from the origin server and populate its cache with this new asset, and then send this new cached asset to the user.

<span style="color: #d4d4d4;">Sites with heavy traffic work well with pull CDNs, as traffic is spread out more evenly with only recently-requested content remaining on the CDN.</span>

&nbsp;

### Push CDNs

<span style="color: #d4d4d4;">Push CDNs receive new content whenever changes occur on the server. And it is server responsibility  <span style="color: #d4d4d4;">for providing content, uploading directly to the CDN, and rewriting URLs to point to the CDN.</span></span>

<span style="color: #d4d4d4;"><span style="color: #d4d4d4;"><span style="color: #d4d4d4;">Content is placed on the CDNs once, instead of being re-pulled at regular intervals.</span></span></span>

&nbsp;

#### **Benefits of Using a CDN**

| **Benefit** | **Explanation** |
| --- | --- |
| **Reduced Latency** | Content is delivered from the nearest edge server, minimizing travel time. |
| **Bandwidth Savings** | Offloads traffic from the origin server, reducing hosting costs. |
| **Scalability** | Handles traffic spikes (e.g., Black Friday sales) without downtime. |
| **Improved Security** | Mitigates DDoS attacks, provides SSL/TLS encryption, and blocks malicious bots. |
| **Global Reach** | Ensures fast performance for users worldwide. |

&nbsp;

#### **CDN Use Cases**

- **Media Streaming**: Netflix uses CDNs to deliver 4K videos without buffering.
    
- **E-Commerce**: Amazon serves product images and checkout pages via CDNs during peak sales.
    
- **Software Updates**: Microsoft distributes Windows updates efficiently.
    
- **Static Websites**: Hosting CSS, JS, and images on a CDN accelerates page loads.
