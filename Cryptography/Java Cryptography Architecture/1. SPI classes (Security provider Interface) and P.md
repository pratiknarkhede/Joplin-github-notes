> SPI is a mechanism that allows a framework or library to provide extensible functionality through a set of interfaces or abstract classes. Service providers implement these interfaces to offer specific services.

&nbsp;

> Service providers register their implementations in a configuration file located in the `META-INF/services` directory of a JAR file. This file contains the fully qualified name of the service provider implementation.

&nbsp;

At runtime, the service loader (e.g., `ServiceLoader` class) reads the `META-INF/services` configuration files and loads the service provider implementations.

&nbsp;

* * *

### Consider `java.security` Package and Provider class

it provides a variety of security-related services, such as cryptographic algorithms and key management.

A **Provider** in Java Security is a class that extends the `java.security.Provider` class. Each provider class defines a set of cryptographic algorithms or services (e.g., message digests, signature algorithms, encryption algorithms, key generation) that it supports.

&nbsp;

When you install a security provider in your Java environment, you essentially add it to a list of available providers. The Java Security API can then use these providers to access the cryptographic algorithms and services you need.

it extends `Properties` class which is subclass of HashTable primarly used storing key value pairs in string format , 

Extending to `Properties` class helps in defining the services provider is offering  
Each service is registered as property and the key is string like `"MessageDigest.SHA-256"` and value is fully qualified class name in the jar of provider `com.example.crypto.SHA256Digest`  

The Java Security API uses these properties to look up the appropriate implementation class for a requested service. For example, if a `MessageDigest` instance for "SHA-256" is requested, the API will look through the registered providers' properties to find the implementation that provides this service

* * *

**Suppose we want to create custom Service provider for the SPI**

```java
import java.security.Provider;

public class MySecurityProvider extends Provider {
    public MySecurityProvider() {
        super("MyProvider", 1.0, "My Security Provider v1.0");

        // Register services
        put("MessageDigest.SHA-256", "com.example.crypto.SHA256Digest");
        put("Cipher.AES", "com.example.crypto.AESCipher");
        put("KeyPairGenerator.RSA", "com.example.crypto.RSAKeyPairGenerator");
    }
}

```

now we will be providing the custom provider as a jar to whichever project wanna use it ,   
But First we would need to register it as Service in the Properties mentioned earlier  
<br/>For that :  
create a file named `java.security.Provider` in the `META-INF/services` directory of  JAR file. The file should contain the fully qualified name of implementation class.

&nbsp;

Now to make sure that instead using default service provider , but our custom provider  
we need to add that into security like this:  
<br/>

```java
import java.security.Provider;
import java.security.Security;

public class SecurityExample {
    public static void main(String[] args) {
        // Load and register the custom provider
        Provider provider = new MySecurityProvider();
        Security.addProvider(provider);
        
        // Use the provider's services
        // For example, create an instance of the algorithm
    }
}

```

- **SPI Interfaces**: Define the services or functionalities to be provided.
- **Service Providers**: Implement the SPI interfaces and register them using configuration files.
- **Service Loader**: Discover and load service providers at runtime.
