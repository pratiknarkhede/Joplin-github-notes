#### **Understanding POM (Project Object Model)**

- **POM Structure**: The heart of any Maven project, managing dependencies, plugins, and project configurations.  
    <br/>

**Basic POM Structure:  
**

```xml
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    <groupId>com.example</groupId>
    <artifactId>my-app</artifactId>
    <version>1.0.0</version>
    <packaging>jar</packaging>
</project>

```

&nbsp;

&nbsp;

- **`<modelVersion>`**: Always `4.0.0`, indicating the POM model version.
- **`<groupId>`**: Identifier for the group (usually corresponds to a package structure).
- **`<artifactId>`**: Name of the project, should be unique within the group.
- **`<version>`**: Defines the version of the project.
- **`<packaging>`**: Defines the build output type (e.g., `jar`, `war`).

- `<project>` : specifies xml schema of pom , to which it needs to adhere to , we cannot just replace &lt;groupid&gt; with &lt;xyz&gt;  
    there are already predefined tags in that schema which it enforces  
    
    * * *
    
      
    ****Parent Definition:  
    ****
    
    ```java
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>3.1.6</version>
    </parent>
    
    ```
    
      
    Inheriting configurations and dependencies from a common parent, such as Spring Boot’s starter parent, to reduce redundancy and maintain consistency across multiple projects.  
    there is super pom which is every pom is child to , if we didn't specify the parent pom ,   
    the super pom is direct considered as parent implicitely

    #### **Dependencies**
    
    ```xml
    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
            <version>3.1.6</version>
        </dependency>
    </dependencies>
    
    ```
    
    **Dependency Management**
    
    ```xml
    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-dependencies</artifactId>
                <version>3.1.6</version>
                <scope>import</scope>
                <type>pom</type>
            </dependency>
        </dependencies>
    </dependencyManagement>
    
    ```
    
      
    Manages versions of dependencies used across multiple modules. Dependencies are declared here and referenced in child POMs without specifying versions.

**Repositories  
<br/>**

```xml
<repositories>
    <repository>
        <id>central</id>
        <url>https://repo.maven.apache.org/maven2</url>
    </repository>
</repositories>

```

Specifies remote repositories to search for dependencies and plugins. The `<repositories>` section is used to define additional repositories besides Maven Central.

#### <span style="color: #000000;">Plugin Management</span>

```xml
<build>
    <pluginManagement>
        <plugins>
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-compiler-plugin</artifactId>
                <version>3.8.1</version>
                <configuration>
                    <source>17</source>
                    <target>17</target>
                </configuration>
            </plugin>
        </plugins>
    </pluginManagement>
</build>

```

&nbsp;

&nbsp;Similar to dependency management but for plugins. This section configures ==plugin defaults that child POMs can inherit.  
<br/>==

- The `build` section configures the build process.
- `plugins` define additional tools or processes to be executed during the build.
- This example configures the compiler plugin to use Java 17.

if we didn't have child poms . we would only specify `<plugin>` tag 

```xml
<build>
  <plugins>
    <plugin>
      <groupId>org.apache.maven.plugins</groupId>
      <artifactId>maven-compiler-plugin</artifactId>
      <version>3.8.1</version>
      <configuration>
        <source>1.8</source>
        <target>1.8</target>
      </configuration>
    </plugin>
  </plugins>
</build>
```

**Distribution Management**

```xml
<distributionManagement>
    <repository>
        <id>internal-repo</id>
        <url>http://repo.example.com/releases</url>
    </repository>
    <snapshotRepository>
        <id>internal-snapshots</id>
        <url>http://repo.example.com/snapshots</url>
    </snapshotRepository>
</distributionManagement>

```

  
Manages how and where the project is distributed. Defines where to deploy artifacts, distinguishing between release and snapshot repositories.   
<br/>**Settings.xml Configuration**: Located in the Maven installation directory (`~/.m2/settings.xml`), this file can define servers, proxies, and authentication details.  
<br/>

```xml
<settings>
    <servers>
        <server>
            <id>internal-repo</id>
            <username>deployer</username>
            <password>password</password>
        </server>
    </servers>
</settings>

```

  
This setup is crucial for private repositories requiring authentication. The `<servers>` section specifies credentials for repositories defined in the POM.

**Profiles  
**

```xml
<profiles>
    <profile>
        <id>development</id>
        <activation>
            <activeByDefault>true</activeByDefault>
        </activation>
        <properties>
            <environment>dev</environment>
        </properties>
    </profile>
</profiles>

```

  
Profiles allow conditional configuration based on the environment, such as different dependency versions, build settings, or plugin configurations.

**Properties  
<br/>**

```xml
<properties>
    <java.version>17</java.version>
    <spring.version>3.1.6</spring.version>
</properties>

```

Centralizes configuration properties to avoid duplication and simplify version management across the POM.
