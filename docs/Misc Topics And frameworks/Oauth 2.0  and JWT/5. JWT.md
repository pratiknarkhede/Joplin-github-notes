A JWT is a compact, self-contained ==way to securely transmit information between parties as a JSON object.== It's often used for:

- **Authentication:** Verify the identity of a user.
- **Authorization:** Determine what a user is allowed to do (their permissions).
- **Information Exchange:** Securely share information between systems.

&nbsp;

**Structure of a JWT**

A JWT consists of three parts separated by dots:

1.  **Header:** Contains information about how the JWT is signed (e.g., the algorithm used).
2.  **Payload:** Contains the "claims" (statements) about an entity (usually, the user) and additional data. This is where permissions are typically included.
3.  **Signature:** A cryptographic hash of the header and payload, used to verify the JWT's authenticity and integrity.

&nbsp;

```java
{
  "header": {
    "alg": "HS256",
    "typ": "JWT"
  },
  "payload": {
    "sub": "1234567890",  // User ID
    "name": "John Doe",
    "permissions": ["read:messages", "write:messages", "delete:messages"]
  },
  "signature": "xxxxxxx"
}

```

&nbsp;

**How JWTs Work with OAuth**

OAuth is an authorization framework (not authentication). It focuses on how applications or users grant access to their resources. Here's a simplified flow:

1.  **User wants to use App A:** App A needs access to data in App B (e.g., the user's profile on a social network).
2.  **App A requests authorization:** App A redirects the user to App B's authorization server.
3.  **User grants permission:** The user logs into App B and approves App A's request.
4.  **App B issues a JWT:** App B's authorization server sends a JWT back to App A. This JWT includes information about the permissions granted.
5.  **App A uses the JWT:** App A includes the JWT in its requests to App B's resource server.
6.  **App B's resource server verifies the JWT:** It checks the signature and the permissions to ensure the request is valid.

&nbsp;

**Is the JWT Converted Back to JSON?**

Yes, the receiving system (e.g., App B's resource server) decodes the JWT to access the payload (including the permissions). However, it doesn't necessarily need to convert it back to a pretty-printed JSON format; the data is readily available as a JSON object for processing.
