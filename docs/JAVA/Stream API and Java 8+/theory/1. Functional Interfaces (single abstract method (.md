**Functional Interfaces:**

These interfaces (like `Supplier`, `BiConsumer`, `BinaryOperator`, `Function`, and `Predicate`) are designed to have a **single abstract method** (SAM). This SAM defines the contract for a particular type of function. For example:

- `Supplier` represents a function that produces a value.  
    <br/>
- `BiConsumer` represents a function that consumes two arguments and produces no result.  
    <br/>
- `BinaryOperator` represents a function that takes two arguments of the same type and returns a result of the same type.  
    <br/>
- `Function` represents a function that transforms one type of value into another.  
    <br/>
- `Predicate` represents a function that tests a condition and returns a boolean.

* * *

**No Direct Implementations (Usually):**

You typically don't create classes that directly implement these interfaces. Instead,

you provide the implementation of the SAM through lambda expressions or method references when they are needed.

**Lambda Expressions as Implementations:**

**Lambda expressions are concise ways to represent the implementation of the SAM of a functional interface.** The Java compiler automatically generates an implementation class for the functional interface based on the lambda expression.

**Method References as Implementations:**

Method references are a shorthand way to refer to an existing method that matches the SAM of a functional interface. This can make your code more readable and concise.

&nbsp;

```java
// Using a lambda expression
Predicate<Integer> isEven = num -> num % 2 == 0;

// Using a method reference
Function<String, String> toUpperCase = String::toUpperCase;
```

**How it Works:**

1.  You use a lambda expression or method reference where a functional interface is expected.
2.  The compiler checks if the signature of the lambda or method reference matches the SAM of the functional interface.
3.  If it matches, the compiler creates an anonymous implementation class for the functional interface at runtime, effectively providing the implementation of the SAM.

* * *

&nbsp;Examples:

**1\. Supplier:**

- `ArrayList::new` - Creates a new empty `ArrayList`.
- `HashSet::new` - Creates a new empty `HashSet`.
- `StringBuilder::new` - Creates a new empty `StringBuilder`.

&nbsp;

```java
Supplier<List<Integer>> listSupplier = ArrayList::new; 
List<Integer> numbers = listSupplier.get(); // Creates an empty ArrayList
```

&nbsp;

`ArrayList::new` itself is not a direct implementation of the `Supplier` interface. However, it can be used as a method reference to provide the functionality of a `Supplier`.

Let me elaborate:

- **Method References:** In Java, method references (`::`) provide a concise way to refer to a method without explicitly writing a lambda expression. When a method reference matches the signature of a functional interface, it can be used as a lambda expression for that interface.
    
- **`ArrayList::new` as Supplier:**
    
    - The `ArrayList::new` method reference essentially means "call the constructor of ArrayList to create a new instance."
        
    - This matches the signature of the `Supplier` interface, which has a single method `get()` that takes no arguments and returns a value.
        
    - Therefore, `ArrayList::new` can act as a `Supplier<ArrayList>`, providing a way to create new `ArrayList` objects.

**BiConsumer:**

- `ArrayList::add` - Adds an element to a list.
- `Map::put` - Adds a key-value pair to a map.

```java
BiConsumer<List<String>, String> addToList = List::add;
List<String> fruits = new ArrayList<>();
addToList.accept(fruits, "apple"); // Adds "apple" to the fruits list
```
