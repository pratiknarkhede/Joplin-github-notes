**Collectors:**

Imagine a toy organizer with different compartments. Collectors help you decide how to sort the toys you take out of the box into these compartments.

For example:

- A collector could sort toys by color (red toys in one compartment, blue in another).
- Another collector could count how many of each type of toy you have (2 cars, 3 balls).

```java
public interface Collector<T, A, R> {
    Supplier<A> supplier();
    BiConsumer<A, T> accumulator();
    BinaryOperator<A> combiner();
    Function<A, R> finisher();
    Set<Characteristics> characteristics();
}
```

- **T:** The type of input elements to the collector.
- **A:** The mutable accumulation type of the collector (often a collection).
- **R:** The final result type of the collector.

**Methods:**

* * *

- `supplier()`: Provides a function to create a new, empty accumulator object.  
    <br/>
- `accumulator()`: Provides a function to incorporate an element into the accumulator.  
    <br/>
- `combiner()`: Provides a function to merge two accumulators (mainly for parallel streams).  
    <br/>
- `finisher()`: Provides a function to transform the accumulator into the final result.  
    <br/>
- `characteristics()`: Returns a set of characteristics indicating how this collector can be used (e.g., concurrent, unordered).

* * *

**Accumulator:**

This is the worker who actually puts the toys in the compartments. Each time you take a toy out of the box, the accumulator follows the collector's instructions and places the toy in the correct compartment.

* * *

**Supplier:**

Think of the supplier as the person who gives you an empty organizer to start with. Before you begin sorting, the supplier provides you with the compartments where you'll put the toys.

```java
public interface Supplier<T> {
    T get();
}
```

### Provides a function to create new objects. Used by the `supplier()` method in `Collector`.

  
<br/><br/>Finisher:

Once you've looked through all the toys and sorted them into the organizer, the finisher might do a final touch-up. For example, the finisher could arrange the toys neatly in each compartment or put labels on the compartments.  
<br/>consider this code snippet where are printing sum of integer array elements

```java
List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5, 6);

int sum = numbers.stream()
                 .collect(Collectors.summingInt(Integer::intValue));

System.out.println("Sum of numbers: " + sum);
```

&nbsp;

- `Collectors.summingInt(Integer::intValue)` is the collector. It says we want to sum up all the numbers.
- The accumulator adds each number to a running total.
- The supplier provides the initial value for the total (which is 0 in this case).
- The finisher doesn't do anything special here because the total is already calculated.
