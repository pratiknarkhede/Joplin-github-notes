### What Does `final` Mean?

The `final` keyword in Java has different meanings depending on the context:

1.  **For Variables** :
    
    - If a variable is declared `final`, its value cannot be changed after it is initialized.
    - For **primitive types** (e.g., `int`, `double`), this means the value itself cannot change.
    - For **reference variables** (e.g., objects), this means the reference to the object cannot change. However, the **state** of the object being referenced can still change.
2.  **For Methods** :
    
    - A `final` method cannot be overridden by subclasses.
3.  **For Classes** :
    
    - A `final` class cannot be subclassed.

### Applying `final` to Objects

When you declare an object as `final`, you're saying that the reference to the object cannot change. However, the internal state of the object (its fields or properties) can still change unless explicitly made immutable.

&nbsp;

#### Example 1: Primitive Type

&nbsp;

```java
final int x = 10;
x = 20; // ERROR: Cannot assign a new value to a final variable
```

&nbsp;

#### Example 2: Reference Variable (Object)

```java
final ReentrantLock lock = new ReentrantLock();
lock = new ReentrantLock(); // ERROR: Cannot reassign a final reference
```

Here, `lock` is a reference to a `ReentrantLock` object. You cannot make `lock` point to a different `ReentrantLock` object because it's `final`.

However, you **can** still modify the internal state of the object:

```java
lock.lock();   // This modifies the internal state of the lock (acquires the lock)
lock.unlock(); // This modifies the internal state of the lock (releases the lock)
```

&nbsp;

<span style="color: #fafafc;">The</span> `final` <span style="color: #fafafc;">keyword does not prevent you from calling methods on the object or modifying its internal state. It only prevents you from reassigning the reference to a different object.</span>

&nbsp;

### When to Use `final` for Objects?

Here are some common scenarios where declaring an object as `final` is appropriate:

1.  **Shared Resources in Multithreading** :
    
    - If an object is shared between threads (e.g., locks, queues, or counters), declare it as `final` to ensure all threads use the same instance.  
        
        ```java
        final BlockingQueue<Integer> queue = new LinkedBlockingQueue<>();
        ```
        
2.  **Constants or Configuration Objects** :
    
    - If an object represents a constant or configuration value that should not change during the program's execution, declare it as `final`.  
        
        ```java
        final Map<String, String> config = Map.of(
            "host", "localhost",
            "port", "8080"
        );
        ```
        
3.  **Immutable Objects** :
    
    - If you want to create an immutable object, declare its fields as `final`. This ensures that the object's state cannot change after construction.
