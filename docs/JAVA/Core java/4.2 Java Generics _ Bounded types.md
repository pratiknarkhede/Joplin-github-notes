### Bounded Types

Bounded types in generics allow you to restrict the types that can be used as arguments for type parameters.

This is useful when you want to enforce that a generic type must be a subtype or supertype of a specific class or implement a particular interface.

&nbsp;

&nbsp;

#### Upper Bounded Wildcards

An upper bound restricts the type to be a specific type or a subclass of that type using the `extends` keyword.

```java
class ClassName<T extends SuperClass> {
    // Class body
}

```

&nbsp;Here, `T` is restricted to be either `SuperClass` or any subclass of `SuperClass`.

```java
class Box<T extends Number> {
    private T value;

    public void setValue(T value) {
        this.value = value;
    }

    public T getValue() {
        return value;
    }
}

public class Main {
    public static void main(String[] args) {
        Box<Integer> intBox = new Box<>();
        intBox.setValue(100);  // Only Numbers or its subclasses allowed
        System.out.println(intBox.getValue());
    }
}

```

&nbsp;

In this example, the `Box` class can only accept `Number` and its subclasses (like `Integer`, `Double`, etc.).

&nbsp;

#### Lower Bounded Wildcards

A lower bound restricts the type to be a specific type or a supertype of that type using the `super` keyword.

```java
void someMethod(List<? super Integer> list) {
    // Code
}

```

&nbsp;

This method accepts a list of `Integer` or its supertype (like `Number` or `Object`).

&nbsp;

```java
public static void addNumbers(List<? super Integer> list) {
    list.add(10);
    list.add(20);
}

```

&nbsp;

* * *

#### Multiple Bounds

You can use multiple bounds to enforce that a type parameter must extend more than one class or interface.

&nbsp;

```java
class ClassName<T extends Class1 & Interface1 & Interface2> {
    // Class body
}

```

&nbsp;

```java
class BoundedType<T extends Number & Comparable<T>> {
    private T value;

    public BoundedType(T value) {
        this.value = value;
    }

    public boolean isGreaterThan(T other) {
        return value.compareTo(other) > 0;
    }
}

```

&nbsp;

In this example, `T` is restricted to `Number` types that also implement the `Comparable` interface.

### Generics in Methods

Generic methods allow you to define methods that can take parameters of generic types. The type parameters are declared before the return type of the method.

&nbsp;

```java
public <T> void someMethod(T param) {
    // Method body
}

```

&nbsp;

```java
public class Utility {
    public static <T> void printArray(T[] array) {
        for (T element : array) {
            System.out.println(element);
        }
    }

    public static void main(String[] args) {
        Integer[] intArray = {1, 2, 3, 4};
        String[] strArray = {"a", "b", "c"};

        Utility.printArray(intArray);
        Utility.printArray(strArray);
    }
}

```

&nbsp;

Here, `printArray` is a generic method that can accept arrays of any type.

&nbsp;

#### Bounded Type Parameters in Methods

Just like with classes, you can use bounded types in generic methods to restrict the types allowed.

&nbsp;

```java
public static <T extends Number> void processNumber(T num) {
    System.out.println(num.doubleValue());
}

```

Here, the method accepts only `Number` or its subclasses, ensuring type safety while processing the number.

### Generics in Enums

While enums in Java don’t support generics directly, you can use generics within methods defined in enums or as part of their constructors.

&nbsp;

```java
enum Operation {
    ADD, SUBTRACT;

    public <T extends Number> double operate(T a, T b) {
        switch (this) {
            case ADD:
                return a.doubleValue() + b.doubleValue();
            case SUBTRACT:
                return a.doubleValue() - b.doubleValue();
            default:
                throw new AssertionError("Unknown operation");
        }
    }
}

public class Main {
    public static void main(String[] args) {
        double result = Operation.ADD.operate(10, 20);
        System.out.println("Result: " + result);
    }
}

```

&nbsp;

In this example, the `Operation` enum defines a generic method `operate` that works with any type that extends `Number`.

### Wildcards in Generics

Wildcards are used in generics to represent an unknown type. The two main types of wildcards are:

1.  **Unbounded Wildcards (`?`)**: Represents any type. used when we are only doing reading with the values and not writing  
    Example:

```java
public void printList(List<?> list) {
    for (Object elem : list) {
        System.out.println(elem);
    }
}

```

### Advantages of Bounded Types

- **Type Safety**: Bounds enforce restrictions, preventing the use of inappropriate types.
- **Reusability**: You can write more flexible and reusable code with the correct use of bounds.
- **Code Clarity**: Proper bounds make the purpose of a generic class or method clearer to the user.
