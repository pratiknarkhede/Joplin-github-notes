# **Foundational Java Functional Programming Patterns**

Java functional programming (using Streams, Lambdas, and `java.util.function`) often relies on a few key patterns. Below are the most common foundational patterns with examples:

---

## **1. Filtering & Transforming Collections**
### **Pattern:**  
Filter elements → Transform (map) → Collect results.

### **Example:**
```java
List<String> names = List.of("Alice", "Bob", "Charlie", "David");

// Filter names longer than 3 chars, convert to uppercase, and collect into a list
List<String> result = names.stream()
    .filter(name -> name.length() > 3)  // Filter
    .map(String::toUpperCase)          // Transform
    .toList();                         // Collect

System.out.println(result); // [ALICE, CHARLIE, DAVID]
```

**Use Cases:**  
- Data cleaning (filtering invalid entries).  
- Converting data formats (e.g., `String` → `Integer`).  

---

## **2. Grouping & Partitioning**
### **Pattern:**  
Group elements by a key or partition into `true`/`false` buckets.

### **Example:**
```java
record Person(String name, int age) {}

List<Person> people = List.of(
    new Person("Alice", 25),
    new Person("Bob", 30),
    new Person("Charlie", 25)
);

// Group by age
Map<Integer, List<Person>> groupedByAge = people.stream()
    .collect(Collectors.groupingBy(Person::age));

System.out.println(groupedByAge); 
// {25=[Person[name=Alice, age=25], Person[name=Charlie, age=25]], 30=[Person[name=Bob, age=30]]}

// Partition into adults (age >= 18) and non-adults
Map<Boolean, List<Person>> partitioned = people.stream()
    .collect(Collectors.partitioningBy(p -> p.age() >= 18));

System.out.println(partitioned);
// {true=[Alice(25), Bob(30), Charlie(25)], false=[]}
```

**Use Cases:**  
- Categorizing data (e.g., grouping orders by customer).  
- Splitting data into valid/invalid sets.  

---

## **3. Reducing & Aggregating**
### **Pattern:**  
Combine elements into a single result (sum, max, concatenation, etc.).

### **Example:**
```java
List<Integer> numbers = List.of(1, 2, 3, 4, 5);

// Sum all numbers
int sum = numbers.stream().reduce(0, Integer::sum);
System.out.println(sum); // 15

// Find max number
Optional<Integer> max = numbers.stream().max(Integer::compare);
System.out.println(max.orElse(0)); // 5

// Join strings with a delimiter
List<String> words = List.of("Java", "Streams", "Are", "Awesome");
String sentence = words.stream().collect(Collectors.joining(" "));
System.out.println(sentence); // "Java Streams Are Awesome"
```

**Use Cases:**  
- Calculating totals (sum, average, max).  
- Building a single result from a collection (e.g., CSV generation).  

---

## **4. Chaining Optional to Avoid Null Checks**
### **Pattern:**  
Use `Optional` to safely handle nullable values in a functional way.

### **Example:**
```java
Optional<String> name = Optional.ofNullable(getNameFromExternalService());

// Transform if present, or return a default
String upperName = name
    .map(String::toUpperCase)
    .orElse("DEFAULT");

System.out.println(upperName); // Either the uppercase name or "DEFAULT"
```

**Use Cases:**  
- Safe data retrieval from APIs/databases.  
- Avoiding `NullPointerException`.  

---

## **5. FlatMap for Nested Collections**
### **Pattern:**  
Flatten nested collections (e.g., `List<List<T>>` → `List<T>`).

### **Example:**
```java
List<List<Integer>> nestedNumbers = List.of(
    List.of(1, 2, 3),
    List.of(4, 5),
    List.of(6, 7, 8)
);

// Flatten into a single list
List<Integer> flatNumbers = nestedNumbers.stream()
    .flatMap(List::stream)
    .toList();

System.out.println(flatNumbers); // [1, 2, 3, 4, 5, 6, 7, 8]
```

**Use Cases:**  
- Processing nested data (e.g., orders with multiple items).  
- Combining multiple streams into one.  

---

## **6. Lazy Evaluation with Infinite Streams**
### **Pattern:**  
Generate streams dynamically (e.g., Fibonacci sequence).

### **Example:**
```java
// Infinite stream of random numbers
Stream<Double> randomNumbers = Stream.generate(Math::random);

// Take first 5
List<Double> firstFiveRandoms = randomNumbers
    .limit(5)
    .toList();

System.out.println(firstFiveRandoms); // [0.12, 0.95, ...] (5 random numbers)
```

**Use Cases:**  
- Generating test data.  
- Implementing mathematical sequences.  

---

## **7. Composing Predicates & Functions**
### **Pattern:**  
Combine multiple conditions (`Predicate`) or transformations (`Function`).

### **Example:**
```java
Predicate<String> startsWithA = s -> s.startsWith("A");
Predicate<String> longerThan3 = s -> s.length() > 3;

// Combine predicates (AND)
Predicate<String> startsWithAAndLong = startsWithA.and(longerThan3);

List<String> filtered = names.stream()
    .filter(startsWithAAndLong)
    .toList();

System.out.println(filtered); // ["Alice"]
```

**Use Cases:**  
- Dynamic filtering (e.g., search with multiple criteria).  
- Reusing conditions across streams.  

---

## **Key Takeaways**
| Pattern | When to Use |
|---------|-------------|
| **Filter-Map-Collect** | Basic data transformation pipelines. |
| **Grouping/Partitioning** | Categorizing data. |
| **Reduction (sum, max, join)** | Aggregating values. |
| **Optional Chaining** | Safe null handling. |
| **FlatMap** | Flattening nested structures. |
| **Infinite Streams** | Generating data lazily. |
| **Predicate/Function Composition** | Reusable and modular conditions. |
