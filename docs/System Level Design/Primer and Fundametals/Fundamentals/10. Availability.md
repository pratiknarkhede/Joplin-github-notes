<span style="color: #d4d4d4;">Availability is the time a system remains operational to perform its required function in a specific period.</span>

<span style="color: #d4d4d4;">It is a simple measure of the percentage of time that a system, service, or machine remains operational under normal conditions.</span>

- **Measured as "9s"** :
    - 99% uptime = **~7.2 hours downtime/year** (unacceptable for critical systems).
    - 99.9% ("three 9s") = **~8.76 hours downtime/year** .
    - 99.99% ("four 9s") = **~52.6 minutes downtime/year** .
    - 99.999% ("five 9s") = **~5.26 minutes downtime/year** .

&nbsp;

### **Common Threats to Availability**

1.  **Hardware/Software Failures** : Server crashes, disk failures, bugs.
2.  **Network Issues** : Latency, partitions, DDoS attacks.
3.  **Overload** : Sudden traffic spikes (e.g., Black Friday sales).
4.  **Maintenance** : Deployments, database migrations.
5.  **Human Error** : Misconfigured servers, accidental deletions.

&nbsp;

### **Strategies to Achieve High Availability**

#### **1\. Redundancy**

- **Server Redundancy** : Deploy multiple instances of your Spring Boot app behind a load balancer (e.g., AWS ALB, Nginx).
- - **Database Redundancy** : Use **replication** (master-slave) or **multi-node clusters** (e.g., Cassandra, CockroachDB).

#### **2\. Failover Mechanisms**

- **Active-Passive** : Standby servers take over if the primary fails.
- **Active-Active** : Traffic is distributed across all servers; if one fails, others handle the load.
    - Example: Use **Kubernetes** to auto-restart failed pods.

#### **3\. Load Balancing**

- Distribute traffic to prevent overload

#### **4\. Caching**

- Reduce database load and improve response times.

#### **5\. Disaster Recovery**

- **Backups** : Regularly snapshot databases (e.g., AWS RDS automated backups).
- **Multi-Region Deployments** : Deploy your Spring Boot app in multiple AWS regions (e.g., us-east-1 and eu-west-1).
