This section explains how to establish relationships among beans defined in a Spring configuration file. It focuses on two primary approaches:

1.  **Direct Method Call Between @Bean Methods** (Wiring).
2.  **Using @Bean Method Parameters** (Auto-Wiring).

The relationships enable one object to delegate tasks to another, reflecting common practices in object-oriented design.

&nbsp;

* * *

**1 Wiring Beans Using Direct Method Calls**

**Definition**: ==Wiring involves explicitly calling one `@Bean` annotated method from another.== **Spring ensures the method returns the same bean instance already in the context, avoiding redundant creation.**

```java
@Configuration
public class ProjectConfig {

  @Bean
  public Parrot parrot() {  
    Parrot p = new Parrot();
    p.setName("Koko");
    return p;
  }

  @Bean
  public Person person() {
    Person p = new Person();
    p.setName("Ella");
    p.setParrot(parrot());  // Direct method call
    return p;
  }
}

```

&nbsp;

- `parrot()` defines a `Parrot` bean named "Koko."
- `person()` defines a `Person` bean named "Ella" and assigns the `Parrot` bean to the `Person` using the direct method call `parrot()`.

##### **Execution Flow**:

1.  Spring creates the `parrot` bean and adds it to the context.
2.  Spring creates the `person` bean. When `person()` calls `parrot()`, Spring fetches the existing `parrot` bean from the context instead of creating a new instance.

* * *

### **2 Auto-Wiring Beans Using Method Parameters**

**Definition**: In auto-wiring, Spring injects a bean into a method parameter by matching the parameter's type with a bean in the context. This method avoids direct method calls and provides better flexibility.

```java
@Configuration
public class ProjectConfig {

  @Bean
  public Parrot parrot() {
    Parrot p = new Parrot();
    p.setName("Koko");
    return p;
  }

  @Bean
  public Person person(Parrot parrot) {  // Parameter injection
    Person p = new Person();
    p.setName("Ella");
    p.setParrot(parrot);
    return p;
  }
}

```

1.  `person()` takes a `Parrot` parameter.
2.  Spring detects the parameter type and injects the `parrot` bean from the context automatically.

&nbsp;

##### **Execution Flow**:

1.  Spring creates the `parrot` bean and stores it in the context.
2.  When calling `person()`, Spring injects the `parrot` bean into the method's parameter.

&nbsp;

**Advantages of Auto-Wiring**:

- Decouples the configuration logic from method calls.
- Works seamlessly with beans defined using annotations like `@Component` or `@Service`.

&nbsp;

### **Key Takeaways from Section 3.1**

1.  Beans can establish relationships in Spring using wiring or auto-wiring approaches.
2.  **Wiring** involves direct method calls, ensuring Spring fetches the existing bean.
3.  **Auto-Wiring** uses method parameters, making it more flexible for different bean types.
4.  Both approaches adhere to Spring's IoC principles, letting Spring manage the lifecycle and dependencies of beans.
