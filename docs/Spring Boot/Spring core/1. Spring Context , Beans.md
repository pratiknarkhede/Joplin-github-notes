#### **What is the Spring Context?**

- **Definition**: The Spring context is the memory space in the application where object instances (beans) are stored for Spring to manage.
- **Purpose**:
    - Allows Spring to control and manage the lifecycle of objects.
    - Enables Spring to provide features like dependency injection, transaction management, and testing.
    - Acts as a bridge between application code and Spring's functionalities.
- **Analogy**: Think of the Spring context as a container where you register the objects you want Spring to manage.

#### **Adding Beans to the Spring Context**

To add beans to the context, you need to register them explicitly. This ensures Spring is aware of their existence.

##### **Ways to Add Beans**:

1.  **Using the @Bean Annotation**
    
    - A method annotated with `@Bean` is executed during Spring's context initialization.
    - The returned object from the method is added as a bean to the context.
        
        ```java
        @Configuration
        public class ProjectConfig {
        
            @Bean
            Parrot parrot() {
                var p = new Parrot();
                p.setName("Koko");
                return p;
            }
        }
        
        ```
        
          
        
        **Steps**:
        
        1.  Annotate the class with `@Configuration` to designate it as a configuration class.
        2.  Define a method that returns the object you want as a bean.
        3.  Annotate the method with `@Bean`.
        
        - The method name becomes the bean name unless explicitly defined.
        - The object returned is added to the Spring context.
    - Example - Accessing the Bean:
        
        ```java
        public class Main {
            public static void main(String[] args) {
                var context = new AnnotationConfigApplicationContext(ProjectConfig.class);
                Parrot p = context.getBean(Parrot.class);
                System.out.println(p.getName());  // Output: Koko
            }
        }
        
        ```
        
2.  **Adding Multiple Beans**:
    
    - You can add multiple beans of the same type to the context by defining multiple `@Bean` methods.
    - Each method should have a unique name.
        
        ```java
        @Configuration
        public class ProjectConfig {
        
            @Bean
            Parrot parrot1() {
                var p = new Parrot();
                p.setName("Koko");
                return p;
            }
        
            @Bean
            Parrot parrot2() {
                var p = new Parrot();
                p.setName("Miki");
                return p;
            }
        }
        
        ```
        
        Use `getBean("beanName", BeanClass.class)` to retrieve a specific instance.
        
3.  **Adding Beans of Different Types**:
    
    - The Spring context can store objects of different types like `String`, `Integer`, etc.
    - These are treated the same as custom objects.  
        
        ```java
        @Configuration
        public class ProjectConfig {
        
            @Bean
            String message() {
                return "Hello, Spring!";
            }
        
            @Bean
            Integer number() {
                return 42;
            }
        }
        
        ```

#### **Comparing Methods for Adding Beans**

Spring provides three main ways to add beans to the context:

1.  **@Bean Annotation**:
    
    - Flexible: Works for both user-defined and library classes.
    - Suitable for adding multiple beans of the same type.
2.  **Stereotype Annotations** (e.g., `@Component`):
    
    - Easier for user-defined classes.
    - Requires less code but less flexible than `@Bean`.
3.  **Programmatic Registration**:
    
    - Used for dynamic or conditional bean registration.
    - Requires more code but offers the most customization.

* * *

#### **Key Takeaways** 

1.  The **Spring context** is essential for Spring’s ability to manage object instances.
2.  The `@Bean` annotation is a powerful and flexible way to add objects to the Spring context.
3.  Multiple beans of the same type can coexist in the context, each identified by a unique name.
4.  Spring context can manage a wide variety of object types, from custom objects to primitives.
