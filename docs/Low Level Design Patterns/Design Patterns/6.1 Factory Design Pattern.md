The Factory design pattern is all about ==**creating objects without exposing the exact creation logic to the client.**==

It provides an interface for creating objects, but lets subclasses decide which class to instantiate.

**Example :**

```java
// This interface defines the method that all notifications will implement.
public interface Notification {
    void send(String message);
}
```

```java
// EmailNotification class that sends notifications via email.
public class EmailNotification implements Notification {
    @Override
    public void send(String message) {
        System.out.println("Email Notification: " + message);
    }
}
```

```java
// SMSNotification class that sends notifications via SMS.
public class SMSNotification implements Notification {
    @Override
    public void send(String message) {
        System.out.println("SMS Notification: " + message);
    }
}
```

```java
// PushNotification class that sends notifications via Push notifications.
public class PushNotification implements Notification {
    @Override
    public void send(String message) {
        System.out.println("Push Notification: " + message);
    }
}
```

&nbsp;

```java
// This factory class will decide which notification type to create based on input.
public class NotificationFactory {
    // Method to create notification objects based on the type provided.
    public Notification createNotification(String type) {
        if (type == null) {
            return null;
        }
        if (type.equalsIgnoreCase("EMAIL")) {
            return new EmailNotification();
        } else if (type.equalsIgnoreCase("SMS")) {
            return new SMSNotification();
        } else if (type.equalsIgnoreCase("PUSH")) {
            return new PushNotification();
        }
        return null;
    }
}
```

Here’s an updated, more practical list of **Factory Pattern usages in Java/Spring**, focusing on modern and commonly encountered scenarios:

* * *

### **1\. Java Collections Framework**

**Factory Method:** `Collections.unmodifiableList()`, `List.of()`, `Set.of()`  
**Why?** Immutable collections are created via factory methods.

```java
List<String> immutableList = List.of("A", "B", "C");  // Factory method
```

* * *

### **2\. Java Time API (`java.time`)**

**Factory Methods:** `LocalDate.now()`, `LocalDate.of()`  
**Why?** Hide complex date initialization logic.

```java
LocalDate date = LocalDate.of(2023, Month.JANUARY, 1);  // Factory method
```

* * *

### **3\. Logging (SLF4J + Logback)**

**Factory Class:** `LoggerFactory` (SLF4J)  
**Why?** Decouples logging implementation (Logback/Log4j2).

```java
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

Logger logger = LoggerFactory.getLogger(MyClass.class);  // Factory method
```

* * *

### **4\. Spring Dependency Injection**

**Factory Class:** `ApplicationContext`  
**Why?** Creates and manages beans dynamically.

```java
// Spring Boot's implicit factory
@Autowired
private MyService myService;  // Injected by Spring's bean factory
```

**Explicit Factory Bean:**

```java
@Configuration
public class MyFactory {
    @Bean
    public MyService myService() {  // Factory method
        return new MyService();
    }
}
```

* * *

### **Key Takeaways for Interviews**

1.  **Factory Methods** (e.g., `List.of()`) are simpler than full **Factory Classes**.
2.  **Spring DI** is the most powerful real-world factory example.
3.  **SLF4J’s `LoggerFactory`** is the gold standard for logging factories.

**Pro Tip:**

- "Factories hide object creation complexity. Spring’s `@Bean` is a factory on steroids!"
