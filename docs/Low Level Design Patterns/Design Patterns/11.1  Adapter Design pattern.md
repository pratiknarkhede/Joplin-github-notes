### **Why Adapter Pattern? (The Problem)**

Imagine we have:

- A **client** that expects data in a specific format (e.g., `JSON`).
- A **legacy class** or third-party library that provides data in an incompatible format (e.g., `XML`).

**Problem:** The client can't directly use the legacy class because their interfaces don't match. Rewriting the legacy code may be risky or impossible.

**Solution:** We introduce an **Adapter**—a middleman that translates the incompatible interface into one the client understands.

* * *

### **Essence of Adapter Pattern**

The Adapter pattern:

1.  **Bridges the gap** between two incompatible interfaces.
2.  **Wraps the adaptee** (the incompatible class) and exposes the expected interface.
3.  Follows the **"composition over inheritance"** principle (usually).

* * *

### **When to Use It?**

1.  **Integrating legacy code** with new systems.
2.  **Using third-party libraries** whose interfaces don't match your needs.
3.  **Unifying multiple classes** with similar functionality but different interfaces.

* * *

### **Types of Adapters**

1.  **Class Adapter** (uses inheritance) – Extends the adaptee class.
2.  **Object Adapter** (uses composition) – Wraps an instance of the adaptee.

We'll focus on the **Object Adapter** (more flexible and preferred).

**Example: Media Player**

Let's model a media player that only plays MP3 files but needs to adapt to play MP4 and AVI files using an adapter.

#### **1\. Target Interface (What the Client Expects)**

```java
public interface MediaPlayer {
    void play(String audioType, String fileName);
}
```

#### **2\. Adaptee (Incompatible Class)**

```java
public class AdvancedMediaPlayer {
    public void playMp4(String fileName) {
        System.out.println("Playing MP4: " + fileName);
    }

    public void playAvi(String fileName) {
        System.out.println("Playing AVI: " + fileName);
    }
}
```

#### **3\. Adapter (Translator)**

**this adapter will have the adaptee i.e AdvancedMediaPlayer** 

**and we will override original play method to make it compatible to play mp4 or avi**

```java
public class MediaAdapter implements MediaPlayer {
    private AdvancedMediaPlayer advancedPlayer;

    public MediaAdapter() {
        this.advancedPlayer = new AdvancedMediaPlayer();
    }

    @Override
    public void play(String audioType, String fileName) {
        if (audioType.equalsIgnoreCase("mp4")) {
            advancedPlayer.playMp4(fileName);
        } else if (audioType.equalsIgnoreCase("avi")) {
            advancedPlayer.playAvi(fileName);
        }
    }
}
```

#### **4\. Client Code**

```java
public class AudioPlayer implements MediaPlayer {
    private MediaAdapter mediaAdapter;

    @Override
    public void play(String audioType, String fileName) {
        if (audioType.equalsIgnoreCase("mp3")) {
            System.out.println("Playing MP3: " + fileName);
        } else if (audioType.equalsIgnoreCase("mp4") || audioType.equalsIgnoreCase("avi")) {
            mediaAdapter = new MediaAdapter();
            mediaAdapter.play(audioType, fileName);
        } else {
            System.out.println("Unsupported format: " + audioType);
        }
    }
}
```

#### **5\. Demo**

```java
public class AdapterDemo {
    public static void main(String[] args) {
        AudioPlayer player = new AudioPlayer();

        player.play("mp3", "song.mp3");
        player.play("mp4", "movie.mp4");
        player.play("avi", "video.avi");
        player.play("wav", "sound.wav"); // Unsupported
    }
}
```

**Output:**

```
Playing MP3: song.mp3
Playing MP4: movie.mp4
Playing AVI: video.avi
Unsupported format: wav
```

* * *

### **Key Takeaways**

1.  **Adapter acts as a bridge** between incompatible interfaces.
2.  **Object Adapter uses composition** (wraps the adaptee).
3.  **Client remains unchanged**—it only sees the target interface.
4.  **New adapters can be added** without modifying existing code (Open/Closed Principle).

* * *

### **Real-World Analogies**

- **Power Adapters:** A travel adapter lets you plug a US device (client) into a European socket (adaptee).
- **Translators:** A translator adapts a speaker's language (adaptee) to the listener's language (client).

* * *

### **When NOT to Use It?**

- If the interfaces are already compatible.
- If you can refactor the adaptee directly (simpler solution).
