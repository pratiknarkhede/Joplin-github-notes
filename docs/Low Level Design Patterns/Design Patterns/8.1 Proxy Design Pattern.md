The proxy design pattern is a structural design pattern, ==**that provides a intermediary acting as interface to another resource or object.**==

This pattern is particularly useful when you want to add an extra layer of control over access to an object, allowing for functionalities such as lazy loading, access control, logging, and more.

* * *

**Common Usages in java:**

1.  **Access Control:** Proxies can restrict access to certain functionalities based on user permissions. For instance, a proxy can allow only admin users to execute sensitive commands.
2.  **Lazy Initialization:** Proxies can delay the creation of expensive objects until they are needed, improving performance and resource management.
3.  **Remote Method Invocation (RMI):** Java's RMI framework uses proxies to facilitate communication between objects in different JVMs, allowing for seamless remote calls.
4.  **Caching:** Proxies can cache results of expensive operations, returning the cached result for subsequent requests to improve efficiency.
5.  **Logging and Monitoring:** Proxies can log requests and responses, providing insights into application behavior without modifying the original object. can be used for pre or post processing calls/requests

* * *

## Example:

&nbsp;

### **1\. Basic Proxy Example (Virtual Proxy - Lazy Loading)**

**Scenario:** Load a heavy `Database` object only when needed.

#### **Step 1: Define the Subject Interface**

```java
public interface Database {
    void query(String sql);
}
```

#### **Step 2: Real Object (Expensive to Create)**

```java
public class RealDatabase implements Database {
    public RealDatabase() {
        System.out.println("Initializing heavy database connection...");
    }

    @Override
    public void query(String sql) {
        System.out.println("Executing query: " + sql);
    }
}
```

#### **Step 3: Proxy (Controls Access)**

```java
public class DatabaseProxy implements Database {
    private RealDatabase realDatabase;  // Lazy initialization

    @Override
    public void query(String sql) {
        if (realDatabase == null) {
            realDatabase = new RealDatabase();  // Create only on first use
        }
        realDatabase.query(sql);  // Delegate to real object
    }
}
```

#### **Step 4: Client Code**

```java
public class Client {
    public static void main(String[] args) {
        Database proxy = new DatabaseProxy();
        
        // Database not initialized yet
        proxy.query("SELECT * FROM users");  // Initializes here
        proxy.query("DROP TABLE users");
    }
}
```

**Output:**

```
Initializing heavy database connection...  
Executing query: SELECT * FROM users  
Executing query: DROP TABLE users  
```

### **2\. Dynamic Proxy (Java Built-in)**

**Use Case:** Log all method calls to a service.

#### **Step 1: Define Service Interface**

```java
public interface PaymentService {
    void pay(int amount);
}
```

#### **Step 2: Real Service**

```java
public class RealPaymentService implements PaymentService {
    @Override
    public void pay(int amount) {
        System.out.println("Processing payment: $" + amount);
    }
}
```

#### **Step 3: Invocation Handler (Proxy Logic)**

```java
import java.lang.reflect.*;

public class LoggingHandler implements InvocationHandler {
    private Object target;

    public LoggingHandler(Object target) {
        this.target = target;
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("Log: Calling " + method.getName() + "()");
        return method.invoke(target, args);  // Delegate to real object
    }
}
```

#### **Step 4: Create Dynamic Proxy**

```java
public class Client {
    public static void main(String[] args) {
        PaymentService realService = new RealPaymentService();
        
        PaymentService proxy = (PaymentService) Proxy.newProxyInstance(
            PaymentService.class.getClassLoader(),
            new Class[] { PaymentService.class },
            new LoggingHandler(realService)
        );
        
        proxy.pay(100);  // Logs + executes
    }
}
```

**Output:**

```
Log: Calling pay()  
Processing payment: $100  
```

### **Key Takeaways**

| Type | Use Case | Implementation |
| --- | --- | --- |
| **Static Proxy** | Lazy loading, access control | Manual proxy class |
| **Dynamic Proxy** | Logging, AOP, caching | `InvocationHandler` + `Proxy.newProxyInstance()` |

**Spring Uses Proxy For:**

- `@Transactional` (AOP proxies)
- `@Cacheable` (Caching proxies)
- Security (`@PreAuthorize`)

**Interview Tip:**

- "Proxies add a layer of control without changing the original object’s code (Open/Closed Principle)."
