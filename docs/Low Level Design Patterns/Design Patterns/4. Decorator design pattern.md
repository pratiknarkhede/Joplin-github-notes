The Decorator pattern is a structural design pattern that ==allows behavior to be added to an individual object, either statically or dynamically, without affecting the behavior of other objects from the same class.== It's used to extend or alter the functionality of objects at runtime without altering their structure.

The Decorator pattern is like wrapping a gift. Imagine you have a simple gift (our basic object), and you want to make it fancier without changing the gift itself.

1.  Basic Concept:
    - You start with a basic object (like our basic pizza).
    - Instead of changing this object directly, you "wrap" it with another object that adds new features.
    - You can keep wrapping it with more objects to add more features.
2.  Real-world Analogy: Think of a basic pizza as a plain cheese pizza. Now, you want to add toppings:
    - Instead of changing the pizza itself, you're adding layers on top.
    - First, you might add pepperoni (one decorator).
    - Then, you might add extra cheese (another decorator).
    - Each addition doesn't change the original pizza; it just adds to it.
3.  Why Use It:
    - It's flexible: You can add or remove features easily.
    - It's clean: You don't have to change existing code to add new features.
    - It's modular: **Each new feature (decorator) is separate and can be used independently.**
4.  When to Use It:
    - When you want to add responsibilities to objects dynamically.
    - When you want to add features without affecting other objects of the same class.
    - When extending a class by subclassing is impractical or impossible.

So,, instead of creating a new subclass for every possible combination of features (like PepperoniCheesePizza, ExtraCheesePizza, etc.), you create "wrapper" classes that each add one feature. You can then combine these wrappers in any way you need.

&nbsp;`// Component interface`

```java
interface Pizza {
    String getDescription();
    double getCost();
}
```

&nbsp;`// Concrete component`

```java
class BasicPizza implements Pizza {
    @Override
    public String getDescription() {
        return "Basic pizza";
    }

    @Override
    public double getCost() {
        return 4.00;
    }
}
```

&nbsp;

`// Decorator abstract class`

```java
abstract class ToppingDecorator implements Pizza {
    protected Pizza pizza;

    public ToppingDecorator(Pizza pizza) {
        this.pizza = pizza;
    }

    @Override
    public String getDescription() {
        return pizza.getDescription();
    }

    @Override
    public double getCost() {
        return pizza.getCost();
    }
}
```

`// Concrete decorators`

```java
class CheeseDecorator extends ToppingDecorator {
    public CheeseDecorator(Pizza pizza) {
        super(pizza);
    }

    @Override
    public String getDescription() {
        return pizza.getDescription() + ", extra cheese";
    }

    @Override
    public double getCost() {
        return pizza.getCost() + 1.50;
    }
}
//////////////////////////////////////////////////////////////////

class PepperoniDecorator extends ToppingDecorator {
    public PepperoniDecorator(Pizza pizza) {
        super(pizza);
    }

    @Override
    public String getDescription() {
        return pizza.getDescription() + ", pepperoni";
    }

    @Override
    public double getCost() {
        return pizza.getCost() + 2.00;
    }
}
```

`Usage:`

```java
public class PizzaShop {
    public static void main(String[] args) {
        Pizza myPizza = new BasicPizza();
        System.out.println(myPizza.getDescription() + " - $" + myPizza.getCost());

        myPizza = new CheeseDecorator(myPizza);
        System.out.println(myPizza.getDescription() + " - $" + myPizza.getCost());

        myPizza = new PepperoniDecorator(myPizza);
        System.out.println(myPizza.getDescription() + " - $" + myPizza.getCost());
    }
}
```
