### **1\. Singleton Pattern**

**Core Idea:**

- Ensures **one instance** per application (e.g., database connection).
- **Spring Twist:** Spring beans are singletons by default, but scoped to the IoC container (not JVM-wide).

**Code Example:**

```java
public class Database {
    private static Database instance;
    
    private Database() {}  // Private constructor
    
    public static Database getInstance() {
        if (instance == null) {
            instance = new Database();  // Lazy initialization
        }
        return instance;
    }
}
```

**How to Break?**

- **Reflection Attack:**
    
    ```java
    Constructor<Database> constructor = Database.class.getDeclaredConstructor();
    constructor.setAccessible(true);
    Database hackedInstance = constructor.newInstance();  // New instance!
    ```
    
    **Fix:** Use `enum` or throw exception in private constructor.

**Spring Use Case:**

```java
@Bean  // Singleton by default
public DataSource dataSource() {
    return new HikariDataSource();
}
```

### **2\. Strategy Pattern**

**Core Idea:**

- **Switch algorithms dynamically** (e.g., payment methods).
- **Spring Twist:** Leverage `@Qualifier` to inject different strategies.

**Code Example:**

```java
interface PaymentStrategy {
    void pay(int amount);
}

class CreditCard implements PaymentStrategy {
    @Override
    public void pay(int amount) {
        System.out.println("Paid " + amount + " via Credit Card");
    }
}

class PaymentService {
    private PaymentStrategy strategy;
    
    public PaymentService(PaymentStrategy strategy) {
        this.strategy = strategy;  // Injected strategy
    }
    
    void processPayment(int amount) {
        strategy.pay(amount);
    }
}
```

**Spring Use Case:**

```java
@Autowired
public PaymentService(@Qualifier("creditCard") PaymentStrategy strategy) {
    this.strategy = strategy;
}
```

### **3\. Observer Pattern**

**Core Idea:**

- **Pub-Sub model** (e.g., event notifications).
- **Spring Twist:** Built-in `ApplicationEventPublisher`.

**Code Example:**

```java
class OrderEvent extends ApplicationEvent {
    public OrderEvent(String message) {
        super(message);
    }
}

@Service
class OrderService {
    @Autowired
    private ApplicationEventPublisher eventPublisher;
    
    void placeOrder() {
        eventPublisher.publishEvent(new OrderEvent("Order placed!"));
    }
}

@Component
class NotificationListener {
    @EventListener
    public void handleOrderEvent(OrderEvent event) {
        System.out.println("Received: " + event.getSource());
    }
}
```

### **4\. Facade Pattern**

**Core Idea:**

- **Simplify complex subsystems** (e.g., microservices aggregation).

**Code Example:**

```java
class ShippingFacade {
    private InventoryService inventory;
    private LogisticsService logistics;
    
    void shipProduct(String productId) {
        if (inventory.isAvailable(productId)) {
            logistics.scheduleDelivery(productId);
        }
    }
}
```

**Spring Use Case:**

```java
@RestController
class OrderController {
    @Autowired
    private OrderFacade orderFacade;  // Hides order-processing complexity
    
    @PostMapping("/order")
    void createOrder() {
        orderFacade.processOrder();
    }
}
```

### **5\. Factory Pattern**

**Core Idea:**

- **Centralize object creation** (e.g., different response formats).

**Code Example:**

```java
class ResponseFactory {
    static Response create(String type) {
        return switch (type) {
            case "JSON" -> new JsonResponse();
            case "XML" -> new XmlResponse();
            default -> throw new IllegalArgumentException("Invalid type");
        };
    }
}
```

**Spring Use Case:**

```java
@Bean
public RestTemplate restTemplate() {
    return new RestTemplateBuilder().build();  // Factory method
}
```

### **Key Interview Insights**

1.  **Singleton:**
    - "Spring singletons are per-container, not per-JVM. Use `@Scope("prototype")` for non-singletons."
2.  **Strategy:**
    - "Avoid conditional logic (`if-else`) by delegating to strategies."
3.  **Observer:**
    - "Spring’s `@EventListener` is a lightweight observer implementation."
4.  **Facade:**
    - "Microservices often use API Gateways as facades."
