- **Inversion of control**(IOC) does just what its name says: it inverts the traditional hierarchy of control for fulfilling object relationships. ==Instead of relying on application code to define how objects relate to each other, relationships are defined by the framework.== As a methodology, IOC introduces consistency and predictability to object relations, but it does require you, as the developer, to give up some fine-grained control.
- **Dependency injection**(DI) is a mechanism where the framework "injects" dependencies into your app. It's the practical implementation of IOC. Dependency injection hinges on[polymorphism](https://www.javaworld.com/article/3033445/java-101-polymorphism-in-java.html), in the sense that it allows the fulfilment of a[reference type](https://www.javaworld.com/article/3172592/type-dependency-in-java-part-1.html)to change based on configurations in the framework. The framework injects variable references rather than having them manually fulfilled in application code.
