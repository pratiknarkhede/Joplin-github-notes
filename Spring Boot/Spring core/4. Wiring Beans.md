#### **Using the `@Autowired` Annotation to Inject Beans**

The `@Autowired` annotation simplifies the establishment of relationships between beans. By annotating a field, constructor, or setter, Spring automatically resolves and injects the required dependency from the context.

There are three main ways to use `@Autowired`:

1.  Field Injection.
2.  Constructor Injection.
3.  Setter Injection.

### **Field Injection**

**Definition**: Using `@Autowired` on a field directly injects a bean into the class.

```java
@Component
public class Parrot {
    private String name = "Koko";

    @Override
    public String toString() {
        return "Parrot : " + name;
    }
}
____________________________________________________________________
@Component
public class Person {
    private String name = "Ella";

    @Autowired
    private Parrot parrot;  // Field Injection

    @Override
    public String toString() {
        return "Person's name: " + name + ", Person's parrot: " + parrot;
    }
}

____________________________________________________________________
@Configuration
@ComponentScan(basePackages = "com.example")
public class ProjectConfig {}
____________________________________________________________________

public class Main {
    public static void main(String[] args) {
        var context = new AnnotationConfigApplicationContext(ProjectConfig.class);
        Person p = context.getBean(Person.class);
        System.out.println(p);
    }
}
____________________________________________________________________

```

console:

```bash
Person's name: Ella, Person's parrot: Parrot : Koko

```

&nbsp;

- Fields cannot be `final`, limiting immutability.
- Dependency injection via fields is harder to test and maintain.

* * *

### **2 Constructor Injection**

**Definition**: Injecting dependencies via a constructor is the most recommended and commonly used approach. This allows the fields to be `final` and ensures immutability.

```java
@Component
public class Person {
    private final Parrot parrot;

    @Autowired  // Constructor Injection
    public Person(Parrot parrot) {
        this.parrot = parrot;
    }

    @Override
    public String toString() {
        return "Person's parrot: " + parrot;
    }
}

```

&nbsp;

&nbsp;When Spring creates the `Person` bean, it automatically injects the `Parrot` bean into the constructor.

**Advantages**:

- Supports immutability.
- Works seamlessly in unit testing by allowing explicit parameter injection.

* * *

### **3 Setter Injection**

**Definition**: Dependencies are injected using a setter method. This is less common in modern development but can be used for optional dependencies.

```java
@Component
public class Person {
    private Parrot parrot;

    @Autowired  // Setter Injection
    public void setParrot(Parrot parrot) {
        this.parrot = parrot;
    }

    @Override
    public String toString() {
        return "Person's parrot: " + parrot;
    }
}

```

&nbsp;

**Limitations**:

- Does not support immutability.
- Rarely used in production due to verbosity and less clarity.
