## What is First-Level Caching?

First-level caching (also called the persistence context cache) is JPA's automatic caching mechanism that operates within a single `EntityManager` instance. It's not something you turn on or off—it's always working for you.

- **Automatic Identity Management**: When you load an entity, JPA remembers it. If you ask for the same entity again (by ID), JPA returns the instance from memory rather than hitting the database.
- **Dirty Checking**: JPA tracks changes to managed entities, so it knows exactly what fields to update when you commit.  
    <br/>
    
    ```java
    @Transactional
    public void giveRaise(Long employeeId, double percentage) {
        Employee emp = entityManager.find(Employee.class, employeeId);
        
        // Modify the entity
        double currentSalary = emp.getSalary();
        double newSalary = currentSalary * (1 + percentage/100);
        emp.setSalary(newSalary);
        
        // No need to call update/save/merge!
        // When transaction commits, changes are detected and SQL UPDATE is generated
    }
    ```
    
- **Repeatable Reads**: You get consistent views of your data within a transaction, even if the database changes underneath.  
    - The first-level cache ensures you get consistent views of your data within a transaction  
        <br/>
        
        ```java
        @Transactional
        public void consistencyExample(Long employeeId) {
            Employee emp1 = entityManager.find(Employee.class, employeeId);
            
            // Even if another transaction updates this employee in the database now,
            // we'll still see our original version within this transaction
            
            Employee emp2 = entityManager.find(Employee.class, employeeId);
            // emp1 and emp2 are identical, regardless of external changes
            
            // To see updates from the database:
            entityManager.refresh(emp1);
        }
        ```
