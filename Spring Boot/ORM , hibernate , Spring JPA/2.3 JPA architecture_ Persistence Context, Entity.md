### **4\. Persistence Context**

The **Persistence Context** is the runtime environment where entity instances are managed by the **`EntityManager`** . It acts as a bridge between the application and the database, ensuring efficient and consistent interaction.

#### **Role of the Persistence Context**

1.  **Acts as a First-Level Cache:**
    
    - The persistence context serves as a **first-level cache** , storing managed entities to reduce database queries.
    - When an entity is retrieved or persisted, it is stored in the persistence context. Subsequent requests for the same entity within the same context return the cached instance instead of querying the database again.
2.  **Tracks Changes to Entities (Dirty Checking):**
    
    - The persistence context automatically tracks changes made to managed entities.
3.  Within a single persistence context, there is only **one instance of an entity** for a given database row.
    
    ```java
    Employee emp1 = em.find(Employee.class, 1L); // Fetches entity from database
    Employee emp2 = em.find(Employee.class, 1L); // Returns the same instance (from the persistence context)
    System.out.println(emp1 == emp2); // true
    ```
    
4.  **Manages Entity State Transitions:**
    
    - The persistence context manages the lifecycle and state transitions of entities:
        - - **New (Transient)** : A newly created entity not yet associated with the persistence context.
                - **Managed** : An entity that is being tracked by the persistence context.
                - **Detached** : An entity no longer associated with the persistence context.
                - **Removed** : An entity scheduled for deletion from the database.
5.  **Can Be Transaction-Scoped or Extended:**
    
    - **Transaction-Scoped** : The persistence context exists only for the duration of a single transaction.
    - **Extended** : The persistence context spans multiple transactions, typically used in stateful applications

EntityManager operations like insert, update , delate are transaction bounded, meaning it first check if transaction is open , if it isnt it throws exception  
<br/>While read operations arent transaction bound

### Transaction Manager:

<span style="color: #d1d5db;">The</span> **Transaction Manager** <span style="color: #d1d5db;">plays a critical role in coordinating transactions across resources (e.g., databases) and synchronizing them with the persistence context.</span>

#### **1\. Transaction Manager and EntityManager:**

<span style="color: #d1d5db;">transactions are managed by the container using</span> **JTA (Java Transaction API)** <span style="color: #d1d5db;">. The</span> `EntityManager` <span style="color: #d1d5db;">works in conjunction with the</span> **Container-Managed Transaction Manager** <span style="color: #d1d5db;">.</span>

- The **container-managed transaction manager** handles transaction boundaries (`begin`, `commit`, `rollback`) automatically.
    
- The `EntityManager` is injected into the application using `@PersistenceContext`
    
    ```JAVA
    @PersistenceContext
    private EntityManager em;
    ```
    
    &nbsp;
    
    - When a method annotated with `@Transactional` (or equivalent) is invoked, the container starts a transaction and associates the persistence context with it.
    - The `EntityManager` interacts with the persistence context to manage entities and synchronize changes with the database.
    - At the end of the transaction, the container commits the transaction and flushes changes to the database.
- The container manages the `EntityManagerFactory` and injects `EntityManager` instances into the application using `@PersistenceUnit` or `@PersistenceContext`.
    
    - - - The `EntityManagerFactory` is typically configured via `persistence.xml` or Spring Boot's `application.properties`.

#### 7\. Dialect

Adapts SQL generation to specific database platforms:

**Role**:

- Encapsulates database-specific SQL syntax
- Handles pagination, locking, sequences
- Determines supported features
- Maps Java types to database-specific type

#### 8\. JPQL (Java Persistence Query Language)

An object-oriented query language similar to SQL but operates on entities:

```java
// JPQL query
TypedQuery<Employee> query = em.createQuery(
    "SELECT e FROM Employee e WHERE e.department = :dept", Employee.class);
query.setParameter("dept", "IT");
List<Employee> employees = query.getResultList();
```

- Provides database-agnostic query capabilities
- Operates on entity model rather than tables
- Supports joins, aggregations, projections
- Can be statically defined or dynamically built
