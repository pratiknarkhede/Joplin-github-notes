A **Step** is an independent phase within a job. Each step typically involves reading, processing, and writing data.

- **Key Concepts** :
    
    - **StepExecution** : Represents the execution of a specific step within a job.
    - **Chunk-Oriented Processing** : Steps process data in chunks (batches) rather than one item at a time.

**Configuration:**

```java
@Bean
public Step processOrdersStep(StepBuilderFactory stepBuilderFactory, ItemReader<Order> reader,
                              ItemProcessor<Order, Invoice> processor, ItemWriter<Invoice> writer) {
    return stepBuilderFactory.get("processOrdersStep")
                             .<Order, Invoice>chunk(10)
                             .reader(reader)
                             .processor(processor)
                             .writer(writer)
                             .build();
}
```

- **Purpose** : Define a `Step` named `processOrdersStep`.
    
- **Details** :
    
    - A `Step` represents an independent phase within a job.
    - Here, we configure the step to process data in chunks of size 10.
    - The `<Order, Invoice>` generic types specify:
        - Input type: `Order` (read from the `ItemReader`).
        - Output type: `Invoice` (written by the `ItemWriter` after processing).
    - The `.reader()`, `.processor()`, and `.writer()` methods wire up the components for reading, processing, and writing data.
    - The `.build()` method finalizes the step configuration.
- **Real-World Example** :
    
    - A step could be "Read Orders from CSV," where you read 100 orders at a time, process them, and write them to a database.
