The **ItemWriter** writes the processed data to a destination (e.g., database, file, API). It writes items in chunks, which improves performance by reducing the number of I/O operations.

- **Common Implementations** :
    
    - `FlatFileItemWriter`: Writes data to flat files.
    - `JdbcBatchItemWriter`: Writes data to a database using JDBC.
    - `JpaItemWriter`: Writes data using JPA.

```java
@Bean
public FlatFileItemWriter<Invoice> writer() {
    FlatFileItemWriter<Invoice> writer = new FlatFileItemWriter<>();
    writer.setResource(new FileSystemResource("invoices.csv"));
    writer.setLineAggregator(new DelimitedLineAggregator<>() {{
        setDelimiter(",");
        setFieldExtractor(new BeanWrapperFieldExtractor<>() {{
            setNames(new String[]{"orderId", "customerId", "amount"});
        }});
    }});
    return writer;
}
```

- - **Purpose** : Define an `ItemWriter` to write `Invoice` objects to a CSV file.
        - **Details** :
            - We use `FlatFileItemWriter` to write data to a flat file (`invoices.csv`).
            - The `.setResource()` method specifies the output file location.
            - The `LineAggregator` converts `Invoice` objects into lines of text:
                - `DelimitedLineAggregator`: Joins fields into a comma-separated string.
                - `BeanWrapperFieldExtractor`: Extracts fields from the `Invoice` object.

<span style="color: #d1d5db;">This component writes processed</span> `Invoice` <span style="color: #d1d5db;">objects to a CSV file.</span>

```csv
orderId,customerId,amount
1,101,100.00
2,102,200.00
```

Overall

Here’s what happens when the job runs:

1.  **Job Execution** :
    
    - The `processOrdersJob` is launched.
    - It starts executing the `processOrdersStep`.
2.  **Step Execution** :
    
    - The `ItemReader` reads 10 `Order` objects from the `orders.csv` file.
    - The `ItemProcessor` transforms each `Order` object into an `Invoice` object.
    - The `ItemWriter` writes the 10 `Invoice` objects to the `invoices.csv` file.
    - This process repeats until all orders are processed.
3.  **Chunk-Oriented Processing** :
    
    - Each chunk of 10 items is processed within a single transaction.
    - If any item in the chunk fails, the entire chunk is rolled back.
