The **ItemReader** is responsible for reading data from a source (e.g., database, file, API). It reads one item at a time and passes it to the processor.

&nbsp;

```java
@Bean
public FlatFileItemReader<Order> reader() {
    FlatFileItemReader<Order> reader = new FlatFileItemReader<>();
    reader.setResource(new ClassPathResource("orders.csv"));
    reader.setLineMapper(new DefaultLineMapper<>() {{
        setLineTokenizer(new DelimitedLineTokenizer() {{
            setNames("orderId", "customerId", "amount");
        }});
        setFieldSetMapper(new BeanWrapperFieldSetMapper<>() {{
            setTargetType(Order.class);
        }});
    }});
    return reader;
}
```

&nbsp;

- **Purpose** : Define an `ItemReader` to read order data from a CSV file.
- **Details** :
    - We use `FlatFileItemReader` to read data from a flat file (`orders.csv`).
    - The `.setResource()` method specifies the file location (in this case, on the classpath). The file `orders.csv` is located in the **classpath** (e.g., `src/main/resources/`).
    - The `LineMapper` converts each line of the file into an `Order` object:
        - `DelimitedLineTokenizer`: Splits each line into fields based on delimiters (e.g., commas).
        - `BeanWrapperFieldSetMapper`: Uses **Spring’s reflection-based mechanism** to automatically map the extracted tokens to the `Order` class’s fields.

```text
orderId,customerId,amount
1,101,100.00
2,102,200.00
```

- **Common Implementations** :
    
    - `FlatFileItemReader`: Reads data from flat files (CSV, TXT).
    - `JdbcCursorItemReader`: Reads data from a database using JDBC.
    - `JpaPagingItemReader`: Reads data using JPA with paging.
