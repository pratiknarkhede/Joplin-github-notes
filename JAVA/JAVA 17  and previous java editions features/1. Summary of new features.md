# Java 17 Features

## Concepts

### Sealed Classes

Sealed classes restrict which other classes or interfaces can extend or implement them. This ensures better control over the class hierarchy.

Real-world example: Suppose you are developing a payment system and want to ensure only specific payment methods are used. You can seal the `PaymentMethod` class to restrict its subclasses to `CreditCard`, `PayPal`, and `BankTransfer`.

```java
public sealed class PaymentMethod permits CreditCard, PayPal, BankTransfer { }

public final class CreditCard extends PaymentMethod { }
public final class PayPal extends PaymentMethod { }
public final class BankTransfer extends PaymentMethod { }
```

### Pattern Matching for instanceof

Pattern matching for `instanceof` simplifies the common pattern of using `instanceof` checks followed by casting.

Real-world example: When processing different shapes in a drawing application, you can directly match and extract the specific type.

```java
if (obj instanceof Circle circle) {
    double radius = circle.getRadius();
    // Process the circle
} else if (obj instanceof Rectangle rectangle) {
    double length = rectangle.getLength();
    double width = rectangle.getWidth();
    // Process the rectangle
}
```

### Records

Records provide a concise way to create classes that are primarily data carriers. They automatically generate boilerplate code like constructors, getters, `equals()`, `hashCode()`, and `toString()`.

Real-world example: Use records to represent simple data structures like a point in a 2D space.

```java
public record Point(int x, int y) { }

Point point = new Point(10, 20);
System.out.println(point.x()); // 10
System.out.println(point.y()); // 20
```

### Text Blocks

Text blocks simplify the inclusion of multi-line string literals in the code, making them more readable and easier to maintain.

Real-world example: When writing JSON or XML within Java code, text blocks make it more readable.

```java
String json = """
{
  "name": "John",
  "age": 30,
  "city": "New York"
}
""";
System.out.println(json);
```

### Enhanced Switch Statements

Enhanced switch statements provide a more concise and flexible way to write switch expressions, supporting both statement and expression forms.

Real-world example: Use an enhanced switch to determine the type of a day in a calendar application.

```java
String dayType = switch (day) {
    case MONDAY, FRIDAY -> "Workday";
    case SATURDAY, SUNDAY -> "Weekend";
    default -> "Unknown";
};
```

### Foreign Function & Memory API (Incubator)

This API provides mechanisms to interact with native code and memory, facilitating tasks that were typically handled by JNI.

Real-world example: Interfacing with a C library for high-performance computations directly from Java.

```java
try (var scope = MemorySession.openConfined()) {
    var intArray = scope.allocateArray(ValueLayout.JAVA_INT, 1, 2, 3);
    // Pass intArray to a native function
}
```

### Vector API (Incubator)

The Vector API allows developers to write complex vector computations that can be optimized at runtime to use hardware acceleration.

Real-world example: Performing large-scale mathematical operations on arrays efficiently.

```java
var a = IntVector.fromArray(IntVector.SPECIES_256, new int[] {1, 2, 3, 4}, 0);
var b = IntVector.fromArray(IntVector.SPECIES_256, new int[] {5, 6, 7, 8}, 0);
var c = a.add(b); // Vector addition
int[] result = new int[4];
c.intoArray(result, 0);
System.out.println(Arrays.toString(result)); // [6, 8, 10, 12]
```
