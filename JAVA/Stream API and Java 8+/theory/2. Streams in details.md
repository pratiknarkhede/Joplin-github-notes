## **What Exactly is a Stream?**

A stream is **not** a data structure ==\- it's a *sequence of elements* supporting functional-style operations on data sources (collections, arrays, I/O).== Think of it as:

1.  **A pipeline** for data processing
2.  **A recipe** for computation (no work happens until terminal operation)
3.  **A view** on underlying data (doesn't store data itself)

## **Key Characteristics**

| Feature | Explanation | Implications |
| --- | --- | --- |
| **Not a data structure** | Doesn't store elements | Can't directly access elements by index |
| **Functional in nature** | Uses lambdas/method references | Enables declarative programming |
| **Lazy evaluation** | Computes only when needed | Enables optimizations like short-circuiting |
| **Consumable** | Can only be traversed once | Like an iterator - must create new stream for reuse |

## **Stream Operations Breakdown**

### **1\. Stream Creation**

How to make streams:

```java
// From collections
List<String> list = Arrays.asList("a", "b", "c");
Stream<String> stream1 = list.stream();

// From arrays
String[] array = {"a", "b", "c"};
Stream<String> stream2 = Arrays.stream(array);

// Static factory methods
Stream<String> stream3 = Stream.of("a", "b", "c");
Stream<Integer> stream4 = Stream.iterate(0, n -> n + 1); // Infinite
Stream<Double> stream5 = Stream.generate(Math::random); // Infinite
```

### **2\. Intermediate Operations (Lazy)**

Return new streams - build the pipeline:

| Operation | Purpose | Example |
| --- | --- | --- |
| `filter(Predicate)` | Select matching elements | `.filter(s -> s.length() > 3)` |
| `map(Function)` | Transform elements | `.map(String::toUpperCase)` |
| `flatMap(Function)` | Flatten nested streams | `.flatMap(list -> list.stream())` |
| `distinct()` | Remove duplicates | `.distinct()` |
| `sorted()` | Sort elements | `.sorted(Comparator.reverseOrder())` |
| `peek(Consumer)` | Debug without modifying | `.peek(System.out::println)` |
| `limit(long)` | Truncate stream | `.limit(5)` |
| `skip(long)` | Skip first N elements | `.skip(2)` |

### **3\. Terminal Operations (Eager)**

Trigger pipeline execution - produce results:

| Operation | Returns | Example |
| --- | --- | --- |
| `forEach(Consumer)` | void | `.forEach(System.out::println)` |
| `collect(Collector)` | Collection/object | `.collect(Collectors.toList())` |
| `toArray()` | Array | `.toArray(String[]::new)` |
| `reduce()` | Single value | `.reduce(0, Integer::sum)` |
| `min()/max()` | Optional | `.max(Comparator.naturalOrder())` |
| `count()` | long | `.count()` |
| `anyMatch()/allMatch()/noneMatch()` | boolean | `.anyMatch(s -> s.contains("a"))` |
| `findFirst()/findAny()` | Optional | `.findFirst()` |

## **Execution Model**

1.  **Setup Phase**: Chain intermediate operations (builds pipeline)
2.  **Execution Phase**: Terminal operation triggers processing
    - Elements flow one-at-a-time through the pipeline
    - No element moves to next operation until previous finishes
    - Short-circuiting possible (e.g., `findFirst()` stops after first match)

```java
// Nothing happens until terminal op
Stream.of("a", "b", "c")
      .filter(s -> {
          System.out.println("filter: " + s);
          return true;
      })
      .forEach(s -> System.out.println("forEach: " + s));

// Output:
// filter: a → forEach: a
// filter: b → forEach: b
// filter: c → forEach: c
```

## **Why Use Streams?**

1.  **Declarative style**: Say *what* not *how*
2.  **Parallelism**: Just add `.parallelStream()`
3.  **Optimization**: Lazy evaluation enables efficiency
4.  **Readability**: Chain operations fluently
5.  **Immutability**: Avoid side-effects

## **Common Pitfalls**

1.  **Reusing streams**: IllegalStateException if reused
2.  **Infinite streams**: Need `limit()` or short-circuiting ops
3.  **Side effects**: Avoid in `filter/map` (use `forEach` instead)
4.  **Null handling**: May need explicit checks

## **When NOT to Use Streams**

- Need direct element access (indices)
- Require multiple passes through data
- Complex branching logic
- Performance-critical sections (sometimes)
