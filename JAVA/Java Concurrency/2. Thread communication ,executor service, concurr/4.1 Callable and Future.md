Think of it this way: you're at a modern coffee shop. You want a fancy latte.

**Callable and Future: Placing Your Order and Getting a Ticket**

Think of `Callable` as your **order to the barista**. It's like saying, "Hey, make me a latte!" The key thing about a `Callable` is that it's a task that **returns a value(latte)**

And in this analogy just like ordering might go wrong , similarly Callable can throw exception.

```java
import java.util.concurrent.*;

public class CallableFutureExample {

    public static void main(String[] args) throws InterruptedException, ExecutionException {
        ExecutorService executor = Executors.newSingleThreadExecutor();

        // Callable represents the order to the barista: "Make me a latte!"
        Callable<String> makeLatteTask = () -> {
            System.out.println("Barista starting to make latte in thread: " + Thread.currentThread().getName());
            Thread.sleep(2000); // Simulate barista taking 2 seconds to make latte
            System.out.println("Barista finished making latte in thread: " + Thread.currentThread().getName());
            return "Latte"; // The barista returns a Latte (String result)
        };

        System.out.println("Placing latte order..."); // Submitting task is like placing the order
        Future<String> latteFuture = executor.submit(makeLatteTask); // Submit the order and get the ticket (Future)

        System.out.println("Order placed, waiting for latte to be prepared (but can do other things)...");

        // ... imagine doing other things here while the latte is being made ...
        // For example, checking emails, reading a book, etc.

        System.out.println("Checking if latte is ready..."); // Checking if the order is done
        if (latteFuture.isDone()) {
            System.out.println("Latte is ready!"); // Order is ready, like order ticket being called
            String latte = latteFuture.get(); // Get the latte using the ticket (Future.get()) - might block!
            System.out.println("Latte received: " + latte); // You got your latte!
        } else {
            System.out.println("Latte is still brewing..."); // Latte is not ready yet
        }

        executor.shutdown(); // Coffee shop closing, no more orders for now
    }
}
```

observe

- we create newSingleThreadExecutor using Executors Factory
- then we created task called makeLatteTask of type Callable  
    - just like Runnable has run() which returns nothing , 
    - Callable has call() which returns result , here expected result is of type string
- then we submitted the Callable task to our executor we created eariler
    - which has return type of Future, which is kind of order ticket in our example,  
        saying you will get your latte ,not now necessarily, go and do other task  
        and check if result is ready 
- then we check if result is ready by using isDone() on future object  
    - if result is ready, we go and retrieve it using latteFuture.get()
- Lastly we make sure to shutdown the executorService

### Handling Timeouts with Future: What if the Barista is Slow?

Now, what if the barista is taking forever? You don't want to stand there waiting indefinitely, right? You need a **timeout**. `Future` gives you that control.

&nbsp;

&nbsp;

```java
public class FutureTimeoutExample {

    public static void main(String[] args) throws InterruptedException, ExecutionException {
        ExecutorService executor = Executors.newSingleThreadExecutor();

        Callable<Integer> slowTask = () -> {
            System.out.println("Starting slow task...");
            Thread.sleep(3000); // Simulate a very slow task
            return 100;
        };

        Future<Integer> future = executor.submit(slowTask);

        try {
            System.out.println("Waiting for result with a timeout...");
            Integer result = future.get(2, TimeUnit.SECONDS); // Wait at most 2 seconds
            System.out.println("Result: " + result);
        } catch (TimeoutException e) {
            System.out.println("Timeout! Task took too long.");
            // Maybe cancel the task if it's still running
            boolean cancelled = future.cancel(true); // Attempt to interrupt the task
            System.out.println("Task cancelled: " + cancelled);
        } finally {
            executor.shutdownNow(); // Best practice to shutdown the executor
        }
    }
}
```

Observe

- we created Callable called slowTask
