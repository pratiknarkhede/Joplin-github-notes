When multiple threads access and modify shared data simultaneously, unexpected behaviors can occur.

To ensure thread safety there are few mechanisms provided by java:

&nbsp;

## Immutable Objects

Immutable objects are perhaps the simplest way to achieve thread safety. Once created, these objects cannot be modified.

&nbsp;

### Why Immutable Objects Are Thread-Safe

When an object can't change, there's no risk of one thread seeing it in an inconsistent state modified by another thread.

```java
package com.pratik.thejavajourney.concurrency.printInorder.thread_creation;

public class ImmutableCartessianPoint {
    private final int x;
    private final int y;
    // silly mistake to avoid
    // if x and y is final they must have initial value
    // initializing private fields at time of declaration is one way
    // but not only one, we can initialize it via constructor also
    public ImmutableCartessianPoint(int x, int y) {
        this.x = x;
        this.y = y;
    }

    public int getX() {
        return x;
    }

    public int getY() {
        return y;
    }

    // no setters since fields are final and we dont want attempt to modify it


    public ImmutableCartessianPoint ImmutableCartessianPoint(int deltaX, int deltaY) {
        return new ImmutableCartessianPoint(x+deltaX,y+deltaY);
    }
}


```

&nbsp;

### Key Characteristics to Create Immutable Objects

1.  Make the class `final` to prevent inheritance
2.  Make all fields `private` and `final`
3.  Don't provide any methods that modify the object's state
4.  Ensure proper construction (all fields initialized in constructor)
5.  Return copies or new objects from methods that would otherwise modify state
